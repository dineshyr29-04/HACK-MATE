# Submission Success Checklist

## [ ] Technical MVP (Minimally Viable Product)
- [ ] Core functionality works without crashing.
- [ ] All API keys/secrets are handled securely (no hardcoding).
- [ ] Code is pushed to a public/private GitHub as per rules.

## [ ] Project Presentation
- [ ] Demo video (usually under 3 minutes) is uploaded.
- [ ] README.md contains clear setup instructions and problem statement.
- [ ] High-quality screenshots or gifs are included in the Devpost/submission page.

## [ ] Final Polish
- [ ] Error states are handled (e.g., "Not found" or "Loading").
- [ ] UI looks consistent and professional.
- [ ] Accessibility check (alt text for images, high contrast).

---
*Tip: Start your submission process 2 hours before the deadline. Something always goes wrong.*
